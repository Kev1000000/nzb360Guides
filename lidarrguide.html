Lidarr Setup Guide

This guide will walk you through the steps to get nzb360 connected to Lidarr running on your computer. If you haven't already, download and install Lidarr on your computer.

Step 1: Primary Connection Address

The first step in connecting nzb360 to Lidarr is filling in your primary connection address into nzb360's Lidarr settings, shown below.

You can find your primary connection address by looking at the URL in your browser on your PC when Lidarr is running.

If your URL shows http://localhost:8686 or http://127.0.0.1:8686 that is not accurate, as it is referring to your localhost address. You will need to find your private IP address for your computer running Lidarr. Follow this guide to find your PRIVATE IP address.

The :8686 is the port number Lidarr is running on. By default, Lidarr uses port 8686. If you haven't changed it, then :8686 is probably what you'll want to use to start.

If you're using a remote address (not a local IP address) then you'll want to use that in your Primary Connection Address.

Once you've entered in your Primary Connection Address, move on to step 2.

Step 2: Adding your API key

An API key is a unique/random key that is given to you by Lidarr that acts as a password to allow apps like nzb360 to communicate securely to it. It must be added to nzb360 exactly as it appears in Lidarr.

To find your specific API key, follow the steps below.

  • With Lidarr launched in your browser, click on the Settings tab on the left.
  • Click the General Tab.
  • Under security, find the API Key field.
  • Copy this key and send it to your mobile device.
  • Paste it in nzb360's Lidarr API Key field.

Note, if you have authentication setup, the API Key will bypass it when using nzb360. You do not need to add your username or password for Lidarr to nzb360.

Step 3: Test your connection

nzb360 allows you to test your connection from the settings screen to see if things will work.

Note: Your mobile device MUST be on the same network as Lidarr (if you're using a local connection, i.e. 192.168.x.x). Make sure your mobile device is on your home wifi.

Press the Test Connection button.

  • If you get a Success you're all set!
  • If you get a Failed to Connect follow the troubleshooting tips below.

Troubleshooting

  • Make sure your Primary Connection String is correct.
  • Double check your API key.
  • Make sure no firewall rules are blocking external access. If on Windows, run Lidarr as Administrator once to auto-add a firewall rule for external connections.
  • Make sure you're on the same network and are connected via wifi on your mobile device.
  • If you're having issues with Quality Profiles or Root Folders not syncing, make sure to add your URL Base field to your primary/local connection strings.

Transmission Setup Guide

This guide will walk you through the steps to get nzb360 connected to Transmission running on your computer. If you haven't already, download and install Transmission on your computer.

Step 1: Choose Transmission as your client

nzb360 supports many different Torrent clients. Make sure to choose Transmission by tapping the row above primary connection string.

Step 2: Set up Remote Access

You must enable remote access in Transmission to allow nzb360 to connect. Follow the steps below to enable within Transmission.

  • Click on Edit -> Preferences.
  • Click on the last tab (Remote)
  • Make sure "Allow remote access" is selected
  • Write down which HTTP port is listed (most likely 9091)
  • It is strongly recommended to add a username and password. Enable the "Use authentication" box and add a username and password that you'll remember.

Note: Keep an eye on the HTTP port in this view. You'll need it the next step.

Step 3: Primary Connection Address

Once you've enabled remote access, you will need to add your primary connection address into nzb360's Torrent's settings, shown below.

NOTE: nzb360 appends /transmission automatically for you. Do not add it to the primary connection string for Transmission, even if you use a reverse proxy/URL rewrite.

To find your primary connection address, follow this guide to find your PRIVATE/LOCAL IP address. It will usually look something like http://192.168.X.X.

Add your IP address to the primary connection string field, along with the HTTP port from the Remote Access settings. Add a colon between the IP address and HTTP port.

Transmission uses port 9091 by default, so your Primary Connection String should look something like http://192.168.X.X:9091

If you're using a remote address (not a local IP address) then you'll want to use that in your Primary Connection Address.

Once you've entered in your Primary Connection Address, move on to step 4.

Step 4: Adding your password

If you added a username/password in Step 2, you will need to enter those within the username/password field in nzb360.

Step 5: Test your connection

nzb360 allows you to test your connection from the settings screen to see if things will work.

Note: Your mobile device MUST be on the same network as Transmission (if you're using a local connection, i.e. 192.168.x.x). Make sure your mobile device is on your home wifi.

Press the Test Connection button.

  • If you get a Success you're all set!
  • If you get a Failed to Connect follow the troubleshooting tips below.

Troubleshooting

  • Make sure your Primary Connection String is correct.
  • Double check your username/password.
  • Try logging into Transmission's Web UI via your browser on your computer by typing in your Primary Connection String exactly. If you can't connect via this method, nzb360 won't work either.
  • Make sure no firewall rules are blocking external access.
  • Make sure you're on the same network and are connected via wifi on your mobile device.
  • Make sure Transmission is running on your PC and remote access is enabled.

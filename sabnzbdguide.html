SABnzbd Setup Guide

This guide will walk you through the steps to get nzb360 connected to SABnzbd running on your computer. If you haven't already, download and install SABnzbd on your computer and configure it to connect to your usenet provider.

Step 1: Primary Connection Address

The first step in connecting nzb360 to SABnzbd is filling in your primary connection address into nzb360's SABnzbd settings, shown below.

You can find your primary connection address by looking at the URL in your browser on your PC when SABnzbd is running. Note, you typically don't want to add /sabnzbd to the end unless you have a URL rewrite. If you don't know what that means, leave off /sabnzbd like in the photo above.

If your URL shows http://127.0.0.1/sabnzbd that is not accurate, as it is referring to your localhost address. You will need to find your private IP address for your computer running SABnzbd. Follow this guide to find your PRIVATE IP address.

The :8080 is the port number SABnzbd is running on. By default, SABnzbd uses port 8080. If you haven't changed it, then :8080 is probably what you'll want to use to start.

If you're using a remote address (not a local IP address) then you'll want to use that in your Primary Connection Address.

Once you've entered in your Primary Connection Address, move on to step 2.

Step 2: Adding your API key

An API key is a unique/random key that is given to you by SABnzbd that acts as a password to allow apps like nzb360 to communicate securely to it. It must be added to nzb360 exactly as it appears in SABnzbd.

To find your specific API key, follow the steps below.

  • With SABnzbd launched in your browser, click on the gear icon on the top right (SABnzbd config).
  • Click the General Tab.
  • Under security, find the API Key field.
  • Copy this key and send it to your mobile device.
  • Paste it in nzb360's SABnzbd API Key field.

While you're in this view, also double check that the very top field (SABnzbd Host) says 0.0.0.0 instead of localhost. If it says localhost, change it to 0.0.0.0, otherwise you won't be able to connect to it.

Step 3: Test your connection

nzb360 allows you to test your connection from the settings screen to see if things will work.

Note: Your mobile device MUST be on the same network as SABnzbd (if you're using a local connection, i.e. 192.168.x.x). Make sure your mobile device is on your home wifi.

Press the Test Connection button.

  • If you get a Success you're all set!
  • If you get a Failed to Connect follow the troubleshooting tips below.

Troubleshooting

  • Make sure your Primary Connection String is correct.
  • Ensure SABnzbd is listening on 0.0.0.0 instead of localhost in SABnzbd's web settings. (Config -> General -> SABnzbd Host)
  • Double check your API key.
  • Make sure no firewall rules are blocking external access.
  • Make sure you're on the same network and are connected via wifi on your mobile device.

Tips and Tricks

  • Did you know you can tap and hold on the Pause button to pause for a certain duration?
  • Did you know you can tap on the Server Info (underneath the 3 dot menu) to quickly access the server info pane?
  • Did you know you can tap on the On Finish label in the Server Info Pane to set the On Finish action?
